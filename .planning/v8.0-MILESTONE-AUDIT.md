## Tech Debt

### Phase 59: Cadence reply-pause hardening (pre-existing)

`processDueCadenceSteps` does not filter REPLIED/CLOSED_LOST sequences from its step query. This is a pre-existing gap from Phase 10, not a Phase 59 regression. The reply-pause works at the quality gate level (quality.ts prevents sends to REPLIED contacts), but the cadence sweep could create unnecessary drafts that would then be blocked at send time. Fix: add `sequence: { status: { notIn: ['REPLIED', 'CLOSED_LOST'] } }` to the `findMany` where clause in engine.ts.

### Phase 56: Legacy CTA constants

CTA_STEP_1/CTA_STEP_2 retained in classifyDraftRisk for legacy template-era drafts still in the database. Can be removed once all legacy drafts have been processed or aged out.
