# Phase 61.3 — Logo Pipeline Unification

**Status:** complete
**Date:** 2026-04-14
**Execution:** lean (no plan-checker / verifier ceremony, single focused session)

## Goal achieved

Collapsed three scattered logo resolution paths into a single `resolveLogoUrl` backend function that HEAD-validates every candidate before persisting. Frontend now trusts the DB column and renders an initial-letter avatar when null — no more browser-side cascade with broken image placeholders.

## What shipped

### Backend — `lib/enrichment/logo-pipeline.ts` (new)

Single entry point `resolveLogoUrl(domain, { apolloLogoUrl? })`. Priority chain:

1. Apollo `logoUrl` — validated with HEAD probe (rejects stale/404 Apollo URLs)
2. Homepage og:image / twitter:image / apple-touch-icon / icon png (via `getHighResLogoUrl`)
3. DuckDuckGo ip3 favicon — only if HEAD probe succeeds (rejects 404-on-miss)
4. Google s2 favicon sz=128 — last resort, HEAD probe catches the 404-with-body case
5. `null` — caller renders an initial-letter avatar

Every HTTP probe uses `AbortSignal.timeout(3000)` and requires `res.ok && res.status === 200 && content-length > 0`. This is the critical fix — previous code accepted 200-with-generic-globe responses silently.

### admin.ts — 3 call sites unified (commit `5f5cb59`)

- `createProspect` IIFE: was `getFaviconUrl` only → now `resolveLogoUrl(cleanDomain)`
- `enrichProspect` no-coverage catch: was `getHighResLogoUrl ?? getFaviconUrl` → now `resolveLogoUrl(domain)`
- `createAndProcess` IIFE: was `getHighResLogoUrl ?? getFaviconUrl` → now `resolveLogoUrl(cleanDomain, { apolloLogoUrl: prospect.logoUrl })`

Direct imports of `getHighResLogoUrl` and `getFaviconUrl` removed from `admin.ts`. The unified pipeline is the only entry point from here on.

### `ProspectLogo` component — 2 stages instead of 4 (commit `bd2bf65`)

Dropped the `apollo → ddg → google → initial` browser-side cascade. New behavior:

- `prospect.logoUrl` is set → render `<img>`
- `<img onError>` fires OR `logoUrl` is null → render initial-letter avatar

The backend guarantee (HEAD-verified URL on write) makes the cascade redundant. Component dropped from 115 to 85 lines. Tests dropped from 11 to 10 cases but cover the simpler state machine more exhaustively.

### Backfill script — `scripts/backfill-logos.ts` (new)

One-off maintenance tool. Walks prospects with empty/null `logoUrl` (or all prospects with `--force`), runs `resolveLogoUrl` against each, writes the result back. Safe to re-run. Dry-run by default.
