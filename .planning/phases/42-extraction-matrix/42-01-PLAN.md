    1. Create `lib/extraction/types.ts` with the stable type contract:
       - `IntentSignal` type: `{ signal: string, confidence: number, sourceUrl: string, snippet: string, sourceType: string }`
       - `IntentCategory` type: one of `'sector_fit' | 'operational_pains' | 'esg_csrd' | 'investment_growth' | 'workforce'`
       - `IntentVariables` type: `{ categories: Record<IntentCategory, IntentSignal[]>, extras: Array<{ category: string, signals: IntentSignal[] }>, populatedCount: number, sparse: boolean }`
       - Export all types for use by Phase 43.

    2. Create `lib/extraction/intent-extractor.ts` with `extractIntentVariables` function:
       - Input: array of evidence items (id, sourceType, sourceUrl, snippet, title, confidenceScore, workflowTag, metadata) + prospect profile (companyName, industry, description, specialties)
       - Filter to quality-gated items only: confidenceScore >= 0.50 (consistent with existing aiRelevance gate)
       - Exclude RAG_DOCUMENT source type items (extraction is from scraped evidence only)
       - Build AI prompt (use Gemini Flash via existing provider abstraction in lib/ai/ if available, otherwise direct Google AI call) that:
         - Receives the filtered evidence snippets with their source URLs and types
         - Instructs the AI to classify each relevant snippet into one or more of the 5 core categories
         - For each extracted signal: provide the signal description, a 0-1 confidence score, the source URL, the backing snippet, and the source type
         - Allow the AI to create additional categories if it finds signals outside the 5 core ones
       - Parse AI response into `IntentVariables` structure
       - Compute `populatedCount` (number of core categories with at least 1 signal)
       - Set `sparse = populatedCount < 3`
       - Return the `IntentVariables` object

       Use the same AI calling pattern as `generateHypothesisDraftsAI` in `lib/workflow-engine.ts` — check that file for the Gemini/Claude call pattern and reuse it. If that uses a provider abstraction, use it. If it calls Gemini directly, do the same.

    3. Add `IntentExtraction` model to `prisma/schema.prisma`:
       ```
       model IntentExtraction {
         id            String      @id @default(cuid())
         createdAt     DateTime    @default(now())
         updatedAt     DateTime    @updatedAt
         variables     Json        // IntentVariables object
         populatedCount Int
         sparse        Boolean     @default(false)
         modelUsed     String?     // which AI model produced this

         researchRunId String      @unique
         researchRun   ResearchRun @relation(fields: [researchRunId], references: [id], onDelete: Cascade)
         prospectId    String
         prospect      Prospect    @relation(fields: [prospectId], references: [id], onDelete: Cascade)

         @@index([prospectId])
         @@index([researchRunId])
       }
       ```
       Add the reverse relations to ResearchRun (`intentExtraction IntentExtraction?`) and Prospect (`intentExtractions IntentExtraction[]`).

Apply schema change: run `npx prisma db push` (or `docker exec` psql if that is the project pattern — check MEMORY.md: "DB drift: apply schema changes via docker exec psql, create migration file manually").
