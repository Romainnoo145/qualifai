Create `lib/signals/detect.ts` exporting the following:

**Types:**

```typescript
export interface DetectSignalsInput {
  previousRunId: string;
  newRunId: string;
  prospectId: string;
  db: PrismaClient;
}

export interface DetectSignalsResult {
  signalsCreated: number;
  skippedByDedup: number;
  signalIds: string[];
}
```

**Constants:**

- `LOOKBACK_DAYS` map: `NEW_JOB_LISTING: 30`, `HEADCOUNT_GROWTH: 60`
- `HEADCOUNT_GROWTH_MIN_DELTA = 5` (minimum absolute increase to fire)
- `HEADCOUNT_GROWTH_MIN_PERCENT = 0.10` (10% minimum growth rate)

**Helper: `normTitle(title: string): string`** — `title.trim().toLowerCase().replace(/\s+/g, ' ')`

**Helper: `extractHeadcount(items: EvidenceItem[]): number | null`**

- First check REGISTRY items for `metadata.werkzamePersonen` (cast metadata as `{ werkzamePersonen?: number } | null`)
- Fallback: parse REGISTRY/LINKEDIN snippet for `/Werkzame personen:\s*(\d+)/`
- Return null if neither found

**Main function: `detectSignalsFromDiff(input: DetectSignalsInput): Promise<DetectSignalsResult>`**

1. Fetch evidence for both runs: `db.evidenceItem.findMany({ where: { researchRunId } })` for both `previousRunId` and `newRunId`
2. **NEW_JOB_LISTING detection:**
   - Filter both arrays for `sourceType === 'CAREERS' || sourceType === 'JOB_BOARD'`
   - Build `Set<string>` of `normTitle(item.title ?? '')` from previous run
   - Find novel jobs from new run where normalized title NOT in previous set
   - If `novelJobs.length > 0`, create ONE SignalCandidate with:
     - `signalType: 'NEW_JOB_LISTING'`
     - `title: \`${novelJobs.length} nieuwe vacature${novelJobs.length > 1 ? 's' : ''} gedetecteerd\``
     - `description: novelJobs.map(j => j.title).join(', ')`
     - `metadata: { count: novelJobs.length, titles: novelJobs.map(j => j.title), newRunId: input.newRunId }`

3. **HEADCOUNT_GROWTH detection:**
   - Call `extractHeadcount()` on both prev and new items
   - Only fire if both are non-null AND `newCount > prevCount`
   - Check thresholds: `delta >= HEADCOUNT_GROWTH_MIN_DELTA || (delta / prevCount) >= HEADCOUNT_GROWTH_MIN_PERCENT`
   - Create ONE SignalCandidate with:
     - `signalType: 'HEADCOUNT_GROWTH'`
     - `title: \`Personeelsgroei: ${prevCount} → ${newCount} (+${delta})\``
     - `description: null`
     - `metadata: { previousCount: prevCount, newCount, delta, newRunId: input.newRunId }`

4. **Lookback dedup** — for each candidate:
   - Query `db.signal.findFirst({ where: { prospectId: input.prospectId, signalType: candidate.signalType, detectedAt: { gte: new Date(Date.now() - LOOKBACK_DAYS[candidate.signalType] * 86400000) } } })`
   - If found, increment `skippedByDedup` counter and skip
   - If not found, create Signal: `db.signal.create({ data: { signalType, title, description, metadata, prospectId: input.prospectId, isProcessed: false } })`

5. Return `{ signalsCreated, skippedByDedup, signalIds }`

Import PrismaClient type from `@prisma/client`. Use `import type` for Prisma types. Do NOT import the prisma singleton — the function receives `db` as parameter for testability.
