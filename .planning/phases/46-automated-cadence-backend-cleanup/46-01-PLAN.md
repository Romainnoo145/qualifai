Modify `processDueCadenceSteps` in lib/cadence/engine.ts:

1. Add import for `generateFollowUp` from `@/lib/ai/generate-outreach` and the necessary context-building helpers. Also add import for PrismaClient types needed to load prospect/hypothesis data for the outreach context.

2. Add two new exported constants at the top of the file alongside the existing types:

   ```
   export const REMINDER_STATUS_OPEN = 'reminder_open';
   export const REMINDER_STATUS_DONE = 'reminder_done';
   export const REMINDER_STATUS_SKIPPED = 'reminder_skipped';
   ```

3. In the `for (const step of dueSteps)` loop, split behavior by channel:

   **For email channel:**
   - Load the prospect's research data (hypotheses, evidence) and previous outreach (last sent subject) to build an OutreachContext object. Use the same pattern as the `generateDraftForContact` function in server/routers/outreach.ts (lines ~375-520) -- load prospect with hypotheses, build the context, call generateFollowUp.
   - Call `generateFollowUp(outreachContext, previousSubject)` to get AI-generated subject + bodyHtml + bodyText.
   - Create OutreachLog with `status: 'draft'` (not 'touch_open'), `type: 'FOLLOW_UP'`, the generated subject/bodyHtml/bodyText, and metadata including `{ kind: 'cadence_draft', createdBy: 'cadence-engine', outreachSequenceId, outreachStepId, evidenceBacked: true }`.
   - This makes the follow-up appear in the existing Drafts Queue (getDecisionInbox queries status='draft').

   **For non-email channels (call, linkedin, whatsapp):**
   - Create OutreachLog with `status: 'reminder_open'` (not 'touch_open'), `type: 'FOLLOW_UP'`, channel set correctly, and metadata including `{ kind: 'cadence_reminder', priority: 'medium', dueAt: step.nextStepReadyAt, createdBy: 'cadence-engine', outreachSequenceId, outreachStepId }`.
   - Use a descriptive subject like `Cadence reminder: {channel} - {companyName}`.

4. Keep the existing step promotion to QUEUED status after creating the OutreachLog (both paths).

5. Wrap the AI generation in try/catch -- if generateFollowUp fails, fall back to creating a draft with empty bodyText/bodyHtml (the admin can manually write copy). Log the error with console.error.

   - Change both occurrences of `status: 'touch_open'` to `status: 'reminder_open'` (the findFirst guard and the create call).
   - Update the metadata kind from `'touch_task'` to `'cadence_reminder'`.
