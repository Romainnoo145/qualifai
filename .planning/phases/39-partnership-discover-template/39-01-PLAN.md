# Phase 39-01 Plan — Atlantis Partnership Trigger Intelligence

## Objective

Introduce an Atlantis-only partnership trigger layer that transforms evidence into concrete partnership signals, with a readiness score and trigger cards visible in Admin Analysis.

## Scope

1. Add deterministic trigger generator + readiness scoring.
2. Persist trigger assessment in research run summary.
3. Expose triggers through hypotheses API for Analysis UI.
4. Keep Klarifai flow unchanged.

## Implementation

1. New module: `lib/partnership/trigger-generator.ts`

- Trigger taxonomy: `strategic_alignment`, `execution_gap`, `commercial_pressure`, `timing_window`
- Output: trigger list, readiness score (0-100), coverage gaps, signal counts
- Placeholder evidence exclusion (`fallback`/`notFound`)

2. Research integration: `lib/research-executor.ts`

- Atlantis-only assessment generation from evidence
- Diagnostics message added
- `runSummaryPayload` extended with `partnership` object

3. API integration: `server/routers/hypotheses.ts`

- `listByProspect` returns `partnership` block from latest run summary for Atlantis

4. UI integration: `components/features/prospects/analysis-section.tsx`

- New Atlantis `Partnership Triggers` block
- Readiness badge + trigger cards (urgency/confidence/evidence)

## Validation

1. `pnpm -s vitest run lib/partnership/trigger-generator.test.ts`
2. `pnpm -s vitest run`
3. `pnpm -s build`
