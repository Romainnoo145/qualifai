2. Extend the `RefreshSweepResult` interface with two new fields (non-breaking, always returned with defaults):
   - `signalsDetected: number;` — sum of signals created across all prospects
   - `draftsCreated: number;` — count from processUnprocessedSignals

3. Inside `runResearchRefreshSweep`, before the `for` loop, declare: `let totalSignalsDetected = 0;`

4. Inside the `for` loop's `try` block, AFTER the `executions.push({ ..., ok: true })` line, add signal detection with its own try/catch:

   ```
   if (candidate.latestRunId) {
     try {
       const signalResult = await detectSignalsFromDiff({
         previousRunId: candidate.latestRunId,
         newRunId: result.run.id,
         prospectId: candidate.prospectId,
         db,
       });
       totalSignalsDetected += signalResult.signalsCreated;
     } catch (signalError) {
       console.error(
         `[research-refresh] signal detection failed for prospect ${candidate.prospectId}:`,
         signalError instanceof Error ? signalError.message : signalError,
       );
     }
   }
   ```

   Key constraints:
   - Only call when `candidate.latestRunId` is non-null (first-time prospects have no baseline to diff)
   - Wrap in its own try/catch — signal detection failure must NOT mark the research execution as failed
   - Place INSIDE the outer try block, after the execution is recorded as ok

5. After the `for` loop (but still inside the `if (!dryRun)` block), call processUnprocessedSignals once:

   ```
   const signalProcessingResult = await processUnprocessedSignals();
   ```

   This processes ALL unprocessed signals at once (not per-prospect). It uses the prisma singleton internally — this is fine for production.

6. For the return statement, add the two new fields:
   - Inside the `if (!dryRun)` path: `signalsDetected: totalSignalsDetected, draftsCreated: signalProcessingResult.draftsCreated`
   - For the dry-run path (when executions array is empty): ensure these default to 0

   Restructure the return to handle both paths. Declare `let draftsCreated = 0;` alongside `totalSignalsDetected`. Inside the `if (!dryRun)` block after calling processUnprocessedSignals, assign `draftsCreated = signalProcessingResult.draftsCreated;`. Then the single return at the bottom always includes `signalsDetected: totalSignalsDetected, draftsCreated`.
