1. **Prisma setup** — Same pattern as tmp-rerun-nedri.ts:

   ```typescript
   import 'dotenv/config';
   import { PrismaPg } from '@prisma/adapter-pg';
   import { PrismaClient } from '@prisma/client';
   import { Pool } from 'pg';
   import { executeResearchRun } from '../lib/research-executor';
   ```

2. **Prospect list** — Look up these 4 prospects by domain/companyName:
   - `stb-kozijnen.nl` (primary success criterion target)
   - `mujjo` (by companyName contains)
   - `nedri.nl` (by domain contains)
   - `marfa.nl` (by domain contains)

3. **Pre-flight check** — For each prospect, query `prisma.prospect.findFirst` with the appropriate filter. If any prospect is not found, log a clear error: `"[PROSPECT_NAME] not found in DB — create via admin UI before running validation"` and skip it (do NOT crash the whole script).

4. **Pre-run evidence count** — Before running each prospect, count existing total EvidenceItems:

   ```typescript
   const preCount = await prisma.evidenceItem.count({
     where: { prospectId: prospect.id },
   });
   ```

5. **Execute pipeline** — For each found prospect, call:

   ```typescript
   const result = await executeResearchRun(prisma as any, {
     prospectId: prospect.id,
     manualUrls: [],
     deepCrawl: true,
   });
   ```

6. **Post-run evidence count per NEW run** — Count items by the new `researchRunId` (NOT total per prospect):

   ```typescript
   const newRunCount = await prisma.evidenceItem.count({
     where: { researchRunId: result.runId },
   });
   ```

7. **Per-sourceType breakdown** for each new run:

   ```typescript
   const breakdown = await prisma.evidenceItem.groupBy({
     by: ['sourceType'],
     where: { researchRunId: result.runId },
     _count: { id: true },
     orderBy: { _count: { id: 'desc' } },
   });
   ```

8. **Stub check** — Count URL-only items (no snippet/title) in the new run:

   ```typescript
   const stubCount = await prisma.evidenceItem.count({
     where: {
       researchRunId: result.runId,
       snippet: null,
       title: null,
     },
   });
   ```

9. **Summary report** — Print a clear table at the end:

   ```
   === V10.0 VALIDATION REPORT ===
   Prospect          | Pre-total | New-run | Stubs | Pass?
   STB-kozijnen      | 233       | 87      | 0     | YES (<100, 0 stubs)
   Mujjo             | 45        | 32      | 0     | YES
   ...
   ```

   Mark STB-kozijnen as PASS if newRunCount < 100 AND stubCount === 0.

10. **Run the script** after creating it: `npx tsx scripts/rerun-v10-validation.ts`
    This will take several minutes per prospect (network calls to Crawl4AI, Gemini Flash scoring, master analysis). Be patient.

11. **After rerun completes**, run the baseline capture script to snapshot the new analyses:
    `npx tsx scripts/capture-baseline.ts`

12. **Run TypeScript check**: `npx tsc --noEmit 2>&1 | head -30`
    The only acceptable error is the pre-existing `lib/enrichment/sitemap.test.ts` Buffer/BodyInit issue. Any NEW errors are a failure.
