## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Tightened generateNarrativeAnalysis + generateKlarifaiNarrativeAnalysis return type to include narrow modelUsed union**

- **Found during:** Task 3 (wrapping research-executor call sites)
- **Issue:** Both functions declared return type as `& { fallbackUsed: boolean }` but omitted `modelUsed` from the intersection. Base `NarrativeAnalysis.modelUsed` is typed as `string`, so `analysisResult.modelUsed` resolved to `string` — causing TS2345 when passing to `recordAnalysisSuccess(db, id, modelUsed)` which expects the narrow `'gemini-2.5-pro' | 'gemini-2.5-flash'` union.
- **Fix:** Added `modelUsed: 'gemini-2.5-pro' | 'gemini-2.5-flash'` to the intersection in both function return types.
- **Files modified:** `lib/analysis/master-analyzer.ts`
- **Verification:** `npx tsc --noEmit` count went from 12 → 10 (back to pre-existing baseline)
- **Committed in:** `83b8f99` (Task 3 commit)
