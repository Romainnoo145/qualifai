### Notes

**POLISH-04 probe order inversion:** REQUIREMENTS.md text specifies "Google primary, DuckDuckGo fallback" but the implementation (after Plan 04's post-smoke-test Rule 1 fix, commit `0cdb4df`) probes DuckDuckGo first, Google second. This inversion is correct â€” DuckDuckGo scrapes real site favicons and returns genuine 404s on misses; Google's s2 returns a generic globe image with HTTP 200 for misses (making client-side `<img onError>` chains unreliable). The requirement's spirit (two-probe favicon fallback) is fully satisfied. REQUIREMENTS.md prose is stale â€” no functional gap.

**`runMasterAnalysis` PRECONDITION_FAILED:** POLISH-07 requires "Run analyse" button. The button exists, fires the mutation, and returns a Dutch-language PRECONDITION_FAILED message ("Analyse herhalen niet ondersteund â€” draai eerst onderzoek opnieuw.") which surfaces via `mapMutationError` in the error slot (confirmed in Romano's smoke test Step 4). The current limitation is architectural: `ProspectAnalysis.inputSnapshot` stores counts only, not reconstructable `NarrativeAnalysisInput`. The design choice is explicit and documented; the requirement does not specify the mutation must succeed, only that the button triggers a mutation and shows feedback.
