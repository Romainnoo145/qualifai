## Decisions Made

- `callGeminiWithRetry` returns `GeminiCallResult` envelope (not raw `GenerateContentResult`) so Plans 03 and 04 can thread `fallbackUsed` end-to-end without re-reading the Prospect row
- Backoff uses `Math.pow(4, attempt) * 1000` = 1s/4s/16s — aggressive enough to wait out Gemini 503 spikes without excessive latency for single-retry cases
- `recordAnalysisFailure` intentionally leaves `lastAnalysisModelUsed` untouched — the previous successful model stays as a history breadcrumb, not overwritten by a failed attempt
- Test for the "throws after primary + fallback exhausted" case uses `expect().rejects` registration BEFORE `runAllTimersAsync` to prevent vitest from seeing an unhandled rejection

## Deviations from Plan

None - plan executed exactly as written. The test file comment `// NO vi.mock('node:timers/promises', ...)` contains the literal string but is not an actual mock call — tests pass cleanly with `vi.useFakeTimers()` only.

- Vitest reported an "unhandled rejection" warning on the `throws with "Failed after"` test when using `await promise` after `runAllTimersAsync`. Fixed by registering `expect().rejects` before advancing timers.
