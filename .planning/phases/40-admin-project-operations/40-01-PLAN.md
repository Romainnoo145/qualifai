# Phase 40-01 Plan — SPV Assignment Plumbing

## Objective

Optional SPV assignment/filtering in scoped Atlantis admin workflows (create, edit, list/filter), while keeping token-scoped project boundaries intact.

## Decision Update (2026-03-07)

This plan is deferred by product decision.

- Reason: added UI controls created noise without clear near-term value.
- Action taken: implementation was rolled back from admin create/list/detail flows.
- Revisit trigger: only if Atlantis operations explicitly require per-prospect SPV assignment in admin UX.

## Scope

1. Backend SPV assignment support

- Extend `admin.createProspect` and `admin.createAndProcess` inputs with optional `spvId`
- Extend `admin.updateProspect` to update/clear `spvId`
- Validate `spvId` belongs to `ctx.projectId` before write

2. Admin UI updates

- New Prospect form: optional SPV selector from `projects.listSpvsForActiveProject`
- Prospect detail: editable SPV selector
- Companies list: SPV chip + SPV filter toggle/dropdown

3. Query ergonomics

- `admin.listProspects` accepts optional `spvId` filter
- Keep all existing project-scoped filters and auth checks

## Validation

1. Create Atlantis prospect with SPV and verify DB linkage
2. Edit existing prospect SPV and verify update
3. List/filter by SPV returns only scoped prospects
4. Ensure Klarifai scope remains unaffected
