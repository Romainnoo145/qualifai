<objective>
Replace keyword-stuffed RAG query construction with AI-driven query generation that analyzes prospect evidence to produce semantically targeted queries.

Purpose: Current queries are built by concatenating profile fragments + hardcoded sector keyword packs (e.g., "housing growth land development infrastructure capacity permits"). This produces generic queries that miss prospect-specific nuances. A steel manufacturer gets the same infrastructure keywords as a construction company. Instead, an LLM should analyze the prospect's actual evidence (83 items for Nedri) and generate 4-6 queries that target what THIS prospect specifically needs from the Atlantis corpus.

Output: `lib/rag/query-builder.ts` module that takes raw evidence items + prospect profile and uses Gemini Flash to generate targeted semantic queries.
</objective>

**Input type:**

```ts
type QueryBuilderInput = {
  companyName: string;
  industry: string | null;
  description: string | null;
  spvName: string | null;
  evidence: Array<{
    sourceType: string;
    workflowTag: string;
    snippet: string;
    confidenceScore: number;
    title: string | null;
  }>;

```

**Output:** `RagQueryInput[]` (same type from retriever.ts — `{ query: string; workflowTag: string }`).

**Implementation:**

1. Import `GoogleGenerativeAI` from `@google/generative-ai` and `GEMINI_MODEL_FLASH` from `@/lib/ai/constants` (use Flash, not Pro — this is a fast utility call, not a master analysis).
2. Check if `GEMINI_MODEL_FLASH` exists in constants. If not, define it as `'gemini-2.5-flash'` in `lib/ai/constants.ts`.
3. Build a concise prompt that:
   - Provides the prospect profile (name, industry, description, SPV context)
   - Provides the TOP 20 evidence items sorted by confidence (trim snippets to 150 chars each to stay within token budget)
   - Asks the LLM to generate exactly 4-6 semantic search queries in Dutch/English mix that would retrieve the most relevant Atlantis partnership documents for THIS prospect
   - Each query should target a different angle: sector capabilities, operational pain relief, compliance/ESG alignment, financial/investment fit, strategic partnership value
   - Format: JSON array of objects with `query` (string, max 120 chars) and `workflowTag` (string)
4. Call Gemini Flash with temperature 0.3 (deterministic, focused).
5. Parse JSON response, validate array of {query, workflowTag} objects.
6. If LLM call fails or returns invalid JSON, return empty array (caller handles fallback).
7. Cap queries at 6 max.

**Prompt design principles:**

- DO NOT include raw keyword packs or sector terms in the prompt
- DO include actual evidence snippets so the LLM understands what this prospect cares about
- DO ask for queries that a human analyst would type to find relevant partnership documentation
- Keep total prompt under 4000 tokens (evidence is the bulk — 20 items × 150 chars = ~3000 chars)

**Lazy init pattern:** Use the same lazy `getGenAI()` pattern from `master-analyzer.ts` with `process.env.GOOGLE_AI_API_KEY`.
