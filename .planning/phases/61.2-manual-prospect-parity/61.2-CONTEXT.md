<decisions>
## Implementation Decisions (locked)

### D1 — Apollo failure handling

- `lib/enrichment/providers/apollo.ts::enrichCompany` MUST catch HTTP 422 responses with "no org found" / "Required parameter" semantics and return a sentinel "no coverage" result instead of throwing
- The downstream `enrichCompany` orchestrator (in `lib/enrichment/index.ts` or wherever the provider chain lives) MUST treat this sentinel as partial success — return a minimal `EnrichedCompanyData` with just `domain` populated, NOT a thrown error
- `admin.enrichProspect` tRPC mutation MUST surface this state as a partial-success amber response, NOT a red error toast
- A new `error-mapping.ts` constant `FRIENDLY_ERROR_APOLLO_NO_COVERAGE` = "Apollo heeft deze organisatie niet in zijn database — verrijking gedeeltelijk." in verbatim Dutch
- The Acties panel `enrichProspect` button MUST render the amber state when this happens, with the Dutch message above
- The pipeline (research → analysis) MUST still be runnable on a partial-success prospect — no blocking on missing Apollo fields

### D2 — Logo source priority

- New helper `lib/enrichment/og-logo.ts` exports `getHighResLogoUrl(domain): Promise<string | null>`
- The helper fetches the homepage HTML (with the existing `lib/scrapling.ts` StealthyFetcher OR plain `fetch` with a 5s `AbortSignal.timeout`), parses for, in priority order:
  1. `<meta property="og:image">` — first match
  2. `<meta name="twitter:image">` — fallback
  3. `<link rel="apple-touch-icon" sizes="...">` — pick the largest size attribute
  4. `<link rel="icon" type="image/png" sizes="...">` — pick the largest size attribute
  5. Returns null if none found
- Must HEAD-probe the chosen URL to confirm it actually returns 200 + a non-zero content-length, otherwise fall through to the next priority
- Must handle relative URLs (`/og-image.png`) and resolve them against the domain
- Wired into `admin.createProspect` as the FIRST step in the fire-and-forget logo enrichment IIFE — replaces the current favicon-only path. Only falls through to favicon if og-logo returns null.
- `ProspectLogo` component's render priority chain becomes: Apollo logoUrl → og-logo (stored in DB by createProspect/enrichProspect) → DDG favicon → Google favicon → initial-letter avatar
- The og-logo URL is stored in `Prospect.logoUrl` on success — same column, different source. No new column needed.

### D3 — Manual prospect inline enrichment form

- `/admin/prospects/new` page gets an "Optionele verrijking" collapsible section below the required fields (domain, slug)
- Optional fields (all nullable, all blank by default):
  - `companyName` — text input
  - `industry` — text input (no dropdown — too many options for a free-form polish phase)
  - `description` — textarea, max 500 chars
  - `employeeRange` — select dropdown with these exact values: `1-10`, `11-50`, `51-200`, `201-500`, `501-1000`, `1001-5000`, `5001+`
  - `city` — text input
  - `country` — text input (default "Nederland")
- `admin.createProspect` mutation accepts these as optional Zod fields and writes them through to Prisma if present
- If `companyName` is set in the form, the fire-and-forget Apollo enrichment IIFE MUST NOT overwrite it (manual entries are sticky — Apollo is the secondary source for manually-entered data)
- The form section is collapsible (default closed) so the create flow stays one-click for the common case

### D4 — Render parity audit

- Walk both `/admin/prospects/[id]` (detail page) and the Evidence + Analysis tabs/pages with a manual prospect that has NULL Apollo fields
- Identify every unguarded access (e.g. `prospect.industry.toLowerCase()` that crashes on null) and add null guards rendering "n/a" or hiding the row
- Add a `<ProspectEnrichmentBadge>` component that renders an amber pill at the top of the detail header reading "Verrijking onvolledig" when `companyName` is null OR `industry` is null OR `description` is null. Single Dutch tooltip explaining what's missing.
- This plan ENDS with a human-verify checkpoint where Romano walks both pages with Marfa and confirms zero crashes + the amber pill renders correctly.

### Claude's Discretion (implementation details)

- Whether to use the existing scrapling service on port 3010 vs plain `fetch` for the og:image scraper — choose based on whether the homepages return useful HTML to plain fetch (test on a few small NL SMB sites first)
- Exact regex vs HTML parser library for og-tag extraction (no new deps preferred — use a simple regex if it's reliable, otherwise minimal hand-written DOM walking)
- Whether `ProspectEnrichmentBadge` is a new file or inlined into the detail page (recommend new file for testability)
- Test coverage shape — match the Phase 61.1 conventions (vitest, mocked fetch, 5-10 cases per pure helper)

</decisions>
