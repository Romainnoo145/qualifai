# Phase 61.2 Plan 02: og-logo Helper Summary

**One-liner:** `getHighResLogoUrl(domain)` regex scraper with 4-priority og:image/twitter:image/apple-touch-icon/icon-png extraction and HEAD-probe confirmation, zero new dependencies.

## What Was Built

`lib/enrichment/og-logo.ts` exports a single async function `getHighResLogoUrl(domain): Promise<string | null>` that:

1. Normalises the domain (strips protocol + path)
2. GETs the homepage with `AbortSignal.timeout(5000)` and a crawler User-Agent
3. Regex-parses the HTML for logo candidates in priority order
4. HEAD-probes each candidate for `200 + non-zero content-length`
5. Returns the first confirmed URL, or `null` if all probes fail

### Extraction Priority Chain

| Priority | Tag                                  | Notes                                                                    |
| -------- | ------------------------------------ | ------------------------------------------------------------------------ |
| 1        | `<meta property="og:image">`         | Both `property=...content=` and `content=...property=` orderings handled |
| 2        | `<meta name="twitter:image">`        | Both attribute orderings handled                                         |
| 3        | `<link rel="apple-touch-icon">`      | Multiple tags allowed; picks largest `sizes=NNxNN` value                 |
| 4        | `<link rel="icon" type="image/png">` | Multiple tags allowed; picks largest `sizes=NNxNN` value                 |

### Regex Patterns Used

```
og:image forward:  /<meta[^>]+property=["']og:image["'][^>]+content=["']([^"']+)["']/i
og:image reverse:  /<meta[^>]+content=["']([^"']+)["'][^>]+property=["']og:image["']/i
twitter forward:   /<meta[^>]+name=["']twitter:image["'][^>]+content=["']([^"']+)["']/i
twitter reverse:   /<meta[^>]+content=["']([^"']+)["'][^>]+name=["']twitter:image["']/i
apple-touch-icon:  /<link[^>]+rel=["']apple-touch-icon["'][^>]*href=["']([^"']+)["'][^>]*/gi
icon/png:          /<link[^>]+rel=["'](?:shortcut )?icon["'][^>]+type=["']image\/png["'][^>]*href=["']([^"']+)["'][^>]*/gi
```

## Test Coverage (8 cases)

1. **og:image happy path** — HTML with `og:image` meta tag, HEAD returns 200 + content-length → returns og:image URL
2. **twitter:image fallback** — og:image HEAD returns 404, twitter:image HEAD returns 200 → returns twitter:image URL
3. **Reversed attribute order** — `content=... property=og:image` ordering → still extracted correctly
4. **Relative URL resolution** — `/og-image.png` in og:image content → resolved to `https://example.com/og-image.png` before HEAD probe
5. **All-fail → null** — all candidates found but all HEAD probes fail → returns null
6. **Timeout/AbortError** — homepage GET throws AbortError → returns null, no HEAD probes attempted
7. **apple-touch-icon with multiple sizes** — three apple-touch-icon links with sizes 57, 180, 120 → picks 180 (largest)
8. **Empty domain → null** — empty string input → returns null without calling fetch

## Deviations from Plan

None — plan executed exactly as written.

The only implementation detail resolved during execution: TypeScript strict mode requires non-null assertions (`!`) on regex capture groups inside `while` loops because the compiler cannot narrow `RegExpExecArray[number]` from `string | undefined` to `string` even when the match index is guaranteed by the regex pattern. Added `!` on `sizeMatch[1]!`, `appleMatch[1]!`, and `iconMatch[1]!`. This is a known TypeScript limitation, not a bug.
