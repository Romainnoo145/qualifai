**Props interface:**

```ts
interface AtlantisDiscoverClientProps {
  companyName: string;
  industry: string | null;
  prospectSlug: string;
  analysis: MasterAnalysis;
  projectBrandName: string;
  bookingUrl: string | null;
  whatsappNumber: string | null;
  phoneNumber: string | null;
  contactEmail: string | null;
  analysisDate: string | null;
}
```

**Wizard infrastructure** — replicate the wizard pattern from `prospect-dashboard-client.tsx`:

- 4 steps with tab labels: `Context`, `Waarom Nu`, `Samenwerking`, `Intake`
- Icons: Sparkles, AlertCircle (or TrendingUp), Lightbulb (or Handshake-equivalent), Phone
- Same step indicators in header with click navigation
- Same prev/next button bar at bottom
- Same AnimatePresence + motion.div swipe animation (variants: enter/center/exit)
- Session tracking: call `api.wizard.startSession`, `api.wizard.trackProgress` with stepTimes — same pattern as DashboardClient
- Brand mark in header (first char of projectBrandName, same styling)

**Tab 0 — Context:**

- Top: a subtle badge/label "Voorstel op maat" (small, muted)
- Headline: `Partnership kansen voor {companyName}` (large, bold)
- Hook subtitle: `analysis.context.hook` rendered as a 1-2 sentence paragraph below headline
- KPI row: 3 KPI blocks in a horizontal row (grid cols-3 on desktop, stacked on mobile)
  - Each KPI: `analysis.context.kpis[i].value` as big bold number (text-3xl or text-4xl font-black), `analysis.context.kpis[i].label` as small label underneath (text-xs uppercase tracking-wide text-gray-500)
  - McKinsey board presentation style — minimal, no borders, no background cards, just the numbers
- Executive hook: `analysis.context.executiveHook` as a paragraph below KPIs with slightly muted text, max-w-prose
- If analysis has no KPIs array or fewer than 3, render what's available gracefully (no crash)

**Tab 1 — Waarom Nu (Triggers):**

- 3 trigger cards in a responsive grid (3 cols desktop, 1 col mobile)
- Each card for `analysis.triggers[i]`:
  - Colored left border accent (4px): use category to pick color:
    - market → blue-ish (`border-l-[#2563EB]`)
    - compliance_esg → green-ish (`border-l-[#059669]`)
    - capital_derisking → amber-ish (`border-l-[#D97706]`)
  - Card header: `trigger.title` (font-semibold, text-sm)
  - Key number: first item from `trigger.numbers[]` displayed prominently (text-xl font-bold). If no numbers, skip this element
  - Narrative: `trigger.narrative` as body text (text-sm text-gray-600)
  - Urgency: subtle indicator — no alarm styling. For high urgency, add a small dot or badge; for medium/low, nothing extra. Keep it boardroom-appropriate.
  - Evidence attribution: Claude's discretion per CONTEXT.md — keep it clean. If trigger has numbers, those serve as implicit attribution. No source URLs or "bron:" labels on the public page.
- Card background: white with subtle shadow (`shadow-sm`), rounded corners
- If fewer than 3 triggers, render what's available

**Tab 2 — Samenwerking (Partnership):**

- Header: "Samenwerkingsroutes" or similar Dutch heading
- List of SPV tracks from `analysis.tracks[]`, each always fully visible (no expand/collapse per CONTEXT.md):
  - Track name: `track.spvName` (font-semibold)
  - Scope: `track.scope` (text-sm text-gray-600)
  - Relevance: `track.relevance` (text-sm, slightly emphasized — italic or different color)
  - Strategic tags: `track.strategicTags[]` as small pills/chips (inline-flex, rounded-full, bg-gray-100 text-xs px-2 py-0.5)
- Below each track (or at bottom of section): a CTA button per track or one general CTA. Per CONTEXT.md, include an interest CTA. Options:
  - "Interesse in {spvName}" button that triggers booking or a simple mailto
  - One general CTA at bottom: "Plan een intake" linking to Cal.com booking
- Use the same Cal.com booking pattern as DashboardClient (getCalApi modal) if bookingUrl is available
- If no tracks in analysis, show a graceful message

**Tab 3 — Intake:**

- Replicate the Contact step from DashboardClient for ATLANTIS context:
  - Cal.com booking button (primary CTA, using same getCalApi pattern)
  - WhatsApp link (if whatsappNumber)
  - Phone link (if phoneNumber)
  - Email link (if contactEmail)
  - Quote request button with confetti (same pattern as DashboardClient)
- The intake tab should look identical to the existing Contact tab in DashboardClient but with Atlantis branding/wording. Reference: DashboardClient lines ~850-1050 (the Contact step rendering).

**Styling guidelines:**

- Same page shell as DashboardClient: `min-h-screen bg-[#F8F9FA] flex flex-col font-sans`
- Same header bar with brand mark, company name, step indicators
- Same content area: `max-w-5xl mx-auto px-6`
- Use CSS variables where DashboardClient does; keep hardcoded brand colors (`#040026`, `#EBCB4B`) consistent
- No generic "bridge" imagery or language per DISC-05
- Analysis date shown subtly in footer or header: "Analyse van {date}" (if analysisDate provided)
- Compact spacing per user preference (p-6 not p-10)

**Dependencies:** Import `motion, AnimatePresence` from framer-motion, `getCalApi` from @calcom/embed-react, lucide icons, `api` from providers, `MasterAnalysis` type from analysis/types.
