**Fix: Dashboard "Start Outreach" button (ISSUE-006)**

The ProspectCard component (~line 260-295) wraps the entire card in a `<Link to="/admin/prospects/${prospect.id}">`. The "Start Outreach" text at ~line 290-292 is a `<span>` inside this Link, so clicking it navigates to prospect detail instead of outreach.

Solution: Convert the span to a `<button>` (or anchor) that stops propagation and navigates to the outreach sub-route:

1. Add `'use client'` is already at top. Import `useRouter` from `next/navigation` if not already imported.
2. Replace the "Start Outreach" span with:

```tsx
<button
  type="button"
  onClick={(e) => {
    e.preventDefault();
    e.stopPropagation();
    router.push(`/admin/prospects/${prospect.id}/outreach`);
  }}
  className="inline-flex items-center gap-2 px-4 py-2 rounded-full text-[10px] font-medium uppercase tracking-[0.08em] bg-gradient-to-b from-[#e4c33c] to-[#f4d95a] text-[var(--color-ink)] border border-[#e4c33c] shrink-0 cursor-pointer hover:brightness-105 transition-all"
>
  Start Outreach
</button>
```

If `router` is not available in ProspectCard scope because it's a sub-component, either:

- Add `const router = useRouter()` inside ProspectCard, or
- Convert ProspectCard to accept an `onOutreach` callback prop and call it from the parent

The key behavior: clicking the gold pill navigates to `/admin/prospects/${prospect.id}/outreach`, NOT to the prospect detail. Clicking anywhere else on the card still navigates to prospect detail.
