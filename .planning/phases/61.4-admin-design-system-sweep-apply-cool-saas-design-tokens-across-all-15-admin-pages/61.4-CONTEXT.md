<decisions>
## Implementation Decisions

### Track 1: Backend Wiring (Critical — 4 issues)

- **"Start voorstel" button** (`app/admin/prospects/[id]/page.tsx:~601`): Wire onClick to navigate to `/admin/prospects/[id]/quotes/new` (create new quote for this prospect)
- **Prospect detail sidebar links** (`app/admin/prospects/[id]/page.tsx`): Evidence, Analyse, Outreach, Resultaten "open" links must navigate to their sub-routes (`/admin/prospects/[id]/evidence`, etc.) — these routes exist on disk but aren't linked
- **Dashboard "START OUTREACH" button**: Currently the entire prospect card is one link. The "START OUTREACH" button should be a separate click target that navigates to `/admin/prospects/[id]/outreach` instead of just the prospect detail
- **Action bar buttons** (Re-enrich, Nieuwe run, Genereer analyse, Start outreach): Verify these are wired to their tRPC mutations. If not, wire them.

### Track 2: Prompt Engineering (Critical — blocking brochure quality)

- **Master prompt restructuring** (`lib/analysis/master-prompt.ts`): Each narrative section must output structured visual data, not just text
- **Type extension** (`lib/analysis/types.ts`): Add `punchline`, `visualType` ("quote" | "comparison" | "signals" | "stats"), and `visualData` to NarrativeSection
- **Brochure rendering** (`components/features/analyse/analyse-brochure.tsx`): Render visual elements based on `visualType` instead of dumping text paragraphs
- **Evidence count fix**: Cover overlay "Bronnen" should use total evidence count, not filtered `evidenceItemsForDiscover`
- **Raw citations**: Hide or style the pipe-separated citation strings at bottom of section pages

### Track 3: Visual/UX Polish (High — 3 issues)

- **Offerte cover text truncated** (`components/features/offerte/` or similar): "AI SYSTEMEN AUTOMAT" is cut off on `/offerte/marfa` cover
- **Offerte missing company name in chrome**: Should show "K x Marfa" like analyse brochure does
- The shared `BrandChrome` component already accepts `companyName` — offerte just needs to pass it
