The BrochureCover component receives `prospect.companyName` via props but never passes it to BrandChrome. There are 7 BrandChrome calls (lines ~132, 315, 526, 769, 1148, 1569, 2056) — ALL without companyName.

**Fix 2: Fix cover text truncation (ISSUE-005)**

The QA report says "AI SYSTEMEN AUTOMAT" is truncated on the cover. Since the cover (page 0) is a video with no text overlay in code, the truncation is likely on page 1 (Uitdaging) or an intro/tagline rendered somewhere.

Investigate the text rendering near the cover. Check:

1. If there's an overlay text on the video cover that uses `overflow: hidden` or constrained dimensions
2. If the Uitdaging page hero text (~line 365-380) has a `maxWidth` that clips long text
3. If the `quote.onderwerp` field is rendered somewhere and gets truncated

The fix depends on what's found:

- If text overflows a container: add `overflow: visible` or remove `maxWidth` constraint, or reduce font-size with clamp for long text
- If text is in the video: this is a video asset issue — add a text overlay on top of the video that renders `quote.onderwerp ?? prospect.companyName` with proper sizing
- If text is in a fixed-height container: increase height or use `min-height` instead of `height`

The cover page's hero statement should use responsive text sizing that doesn't clip. If the hero h1 uses a very large `clamp(56px, 6.5vw, 96px)`, ensure the container has enough vertical space and the text wraps instead of overflowing.

Key: find the element that renders "AI SYSTEMEN AUTOMAT..." (this is likely the `onderwerp` field from the quote data for Marfa), ensure it renders fully visible. If it's a long string like "AI Systemen & Automatisering", the container or font size needs to accommodate it.
