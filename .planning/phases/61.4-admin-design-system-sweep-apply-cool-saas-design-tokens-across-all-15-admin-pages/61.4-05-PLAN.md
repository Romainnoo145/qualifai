  <action>
    Apply the same systematic token mapping to all 6 feature component files:

    **Color substitutions:**
    - `bg-slate-*` → `bg-[var(--color-surface-2)]` or `bg-[var(--color-surface)]`
    - `text-slate-*` / `text-gray-*` → `text-[var(--color-muted)]` or `text-[var(--color-ink)]` depending on shade
    - `border-slate-*` → `border-[var(--color-border)]` or `border-[var(--color-border-strong)]`
    - `text-[#040026]` → `text-[var(--color-ink)]`
    - `bg-emerald-*` / `bg-blue-*` / `bg-purple-*` / `bg-amber-*` / `bg-red-*` → `admin-state-*` classes

    **For quality-chip.tsx and pipeline-chip.tsx specifically:**
    These use color-coded backgrounds to indicate quality/pipeline state. Replace the hardcoded semantic Tailwind colors with the `admin-state-*` CSS classes:
    - Green (good quality / ready) → `admin-state-success`
    - Amber (medium / in progress) → `admin-state-warning`
    - Red (low quality / blocked) → `admin-state-danger`
    - Blue (info state) → `admin-state-info`
    - Use the `admin-state-pill` base class

    **For prospect-actions-panel.tsx:**
    - Replace inline button styles with `admin-btn-primary`, `admin-btn-secondary`, `admin-btn-danger` CSS classes
    - Replace error state red colors with `text-[var(--color-brand-danger)]`
    - Replace success state green with `text-[var(--color-brand-success)]`

    **For quote-form.tsx:**
    - Replace `bg-slate-*` label backgrounds with `bg-[var(--color-surface-2)]`
    - Replace `text-slate-*` labels with `text-[var(--color-muted)]`
    - Ensure form inputs use `input-minimal` class

    **Typography + cleanup (all files):**
    - `font-black` → `font-bold`
    - Remove `ui-tap`, `ui-focus`
    - `rounded-xl` → `rounded-[var(--radius-md)]`

    Keep all component APIs and functionality identical.

  </action>
