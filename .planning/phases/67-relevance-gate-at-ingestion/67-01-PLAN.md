3. Extract the gate logic into a testable pure function exported from `lib/research-executor.ts` so it can be unit-tested without DB mocking:

```typescript
/**
 * Returns true if the item should be KEPT (passes the gate).
 * Returns false if the item should be DROPPED.
 * Items with no aiScore always pass (scorer failure fallback).
 */
export function passesRelevanceGate(
  aiScore: { aiRelevance: number } | undefined,
  sourceType: string,
): boolean {
  if (!aiScore) return true; // No score = pass through
  const threshold =
    RELEVANCE_THRESHOLDS[sourceType] ?? DEFAULT_RELEVANCE_THRESHOLD;
  return aiScore.aiRelevance >= threshold;
}
```

Then in the insert loop, replace the inline gate with: `if (aiScore && !passesRelevanceGate(aiScore, draft.sourceType as string)) { continue; }`
